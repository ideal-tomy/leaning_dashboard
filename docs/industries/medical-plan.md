# 医療・介護業種 横展開計画（初版）

## 体験コンセプト
- 一言で言うと: 配置調整、記録対応、資格・研修管理まで見通せる医療・介護運営デモ
- 管理アプリとしての価値: スタッフ配置、利用者/患者対応、記録不備、研修状況をまとめて見せる
- AI/技術の価値: 配置理由、記録支援、FAQ、言語支援の価値を丁寧に見せる

## まず検討するページ
- ダッシュ: シフト不足、記録未完了、研修期限、重点対応対象を見せる
- 候補者一覧: 資格、経験年数、対応領域、夜勤可否、研修状況を比べられるようにする
- 案件一覧: 配置先、必要資格、勤務条件、ケア特性を示す
- マッチング: 資格・経験・ケア適性・勤務条件の一致理由を説明する
- 書類: 資格証、研修記録、契約・同意関連、帳票の不足を見せる
- ナレッジ: 介護記録、感染対策、ヒヤリハット、利用者対応 FAQ を中心にする
- 実務: 当日引継ぎ、記録確認、申し送りの流れを見せる

## タイトル・カード方針
- `dashboardTitle`: 「医療・介護ダッシュボード」をベースに、安心感と運営精度を感じるコピーにする
- `labels.candidate`: スタッフ候補 / スタッフ のどちらが自然か確認する
- `labels.client`: 配置先 / 勤務先 / ケア案件 のどれが実態に近いか整理する
- 一覧カードの主タイトル:
  - 候補者一覧: 氏名 + 資格 + 対応領域
  - 配置先一覧: 施設名 + シフト条件 + 要件

## 下層ページの中身方針
- 候補者詳細: 資格、夜勤経験、研修履歴、対応可能ケア、コミュニケーション特性を上位表示
- 配置先詳細: 勤務条件、必要スキル、申し送りの注意点、利用者属性のサマリーを整理
- 書類ページ: 記録様式、提出期限、研修関連書類、監査で見られる項目を優先表示
- ナレッジページ: 感染対策、記録ルール、家族対応、事故報告の FAQ を主軸にする

## 次アクション導線
- ダッシュ → 候補者一覧 / 配置先一覧 → マッチング → 書類 / 実務
- マッチング → 配置調整 → 必要記録や研修確認
- 実務 → 当日引継ぎ → 記録未完了フォロー
- モバイルでは「今すぐ確認が必要な配置・記録」が最短で分かることを優先する

## 技術デモの活用
- ナレッジAI: 感染対策、記録ルール、家族対応 FAQ の支援
- マッチングAI: 資格とケア特性に基づく配置理由の説明
- 翻訳: 外国籍スタッフや利用者家族とのコミュニケーション補助
- PDF: 記録様式や提出パックの下書きイメージ
- OCR: 資格証や書類読み取りの補助

## レビュー観点
- 業種担当者: 配置判断と監査対応の両方に必要な情報が見えるか
- Web担当者: 「人手不足対応」と「運営品質向上」が両立して訴求できるか
- UI/UX: 重要情報の優先度が明確で、緊急対応画面としても使えるか

## `staffing` への影響を出さない条件
- この計画は `medical` 向けの配置、記録、研修、FAQ の業種化に限定する
- 派遣コックピットの既存構成は基準として保持し、変更しない
- 共通ファイル変更時は `medical` 条件で閉じるか、`staffing` 既定値を維持する
