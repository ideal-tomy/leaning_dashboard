# 派遣業種 基準メモ

## このファイルの役割
現在のデモで最も作り込まれている `staffing` を、他業種へ横展開する際の基準として使う。

## このファイルを使う理由
- 他業種化の実装で、完成済みの派遣コックピットを崩さないため
- 「どこまで共通化してよいか」「何を別業種だけで閉じるべきか」を判断するため
- Cursor に実装PLANを作らせるときの非変更条件を明文化するため

## 派遣デモの強み
- 候補者、案件、書類、マッチング、実務、ナレッジの流れがつながっている
- 管理アプリとしての安心感と、AI デモの見せ場が分離されている
- モバイル/PC それぞれで体験が整理されている

## 他業種へ持っていくべき型
- 一覧カードで比較軸が明確
- 詳細で判断材料が増える
- 次アクションが自然につながる
- 技術デモが本編の補強として機能する

## 他業種化で必ず変えるべき点
- 用語: 候補者 / 案件 / 書類 / 進捗 の呼び方
- KPI: その業種で重要な数字
- 判断材料: マッチング理由、カード属性、詳細ページの上位情報
- FAQ: 現場が本当に聞く質問
- 実務導線: 1日の業務に合う「次へ」

## 他業種化で原則変えない点
- `staffing` 自体の画面タイトル、カード文言、導線構造
- 既存の技術デモの役割分離
- 既に完成しているモバイル / PC の情報優先設計

## 実装PLANに必ず入れる注意書き
- 対象は該当業種のみ
- `staffing` の文言・レイアウト・導線は変更しない
- 共通ファイルを変更する場合は、対象業種だけに効く条件分岐または既定値維持で閉じる
