# 教育業種 横展開計画（初版）

## 体験コンセプト
- 一言で言うと: 受講者管理、講座進行、提出物、問い合わせ対応を見通しよく進める教育運営デモ
- 管理アプリとしての価値: 受講者、講座案件、進行状況、提出物、問い合わせを統合的に見せる
- AI/技術の価値: FAQ 回答、受講案内、講座推薦理由、問い合わせ補助を見せる

## まず検討するページ
- ダッシュ: 受講状況、未提出物、問い合わせ件数、次回講座を見せる
- 受講者一覧: 進捗、受講意欲、提出状況、継続可能性をカードで比較する
- 講座一覧: 講座テーマ、対象者、開講日、定員、進捗を見せる
- マッチング: 受講者に合う講座やフォロー対象の理由を説明する
- 書類: 申込書、同意書、提出物、修了関連書類の進捗を見せる
- ナレッジ: 受講FAQ、割引、欠席連絡、再受講ルールを主軸にする
- 実務: 開講準備、欠席対応、提出物確認、フォロー連絡を見せる

## タイトル・カード方針
- `labels.candidate` は受講者、`labels.client` は講座案件 or 講座 のどちらが適切か確認する
- 受講者カードでは「受講進捗」「提出状況」「フォロー必要度」を主役にする
- 講座カードでは「講座テーマ」「対象」「空席/定員」を主役にする

## 下層ページの中身方針
- 受講者詳細: 受講履歴、理解度、提出物、フォロー履歴を上位表示
- 講座詳細: 対象受講者、ゴール、教材、次回アクションを整理
- 書類ページ: 申込、同意、修了に関わる必要書類を分かりやすくする
- ナレッジページ: 「再受講」「提出遅延」「欠席」「割引」など運営で頻出の質問を中心にする

## 次アクション導線
- ダッシュ → 受講者一覧 / 講座一覧 → マッチング → 実務 / 書類
- ナレッジ → 問い合わせ対応 → フォロー連絡
- モバイルでは本日のフォロー対象者や未提出者がすぐ分かることを優先する

## 技術デモの活用
- ナレッジAI: 受講FAQ、運営ルール、問い合わせ対応
- マッチングAI: 講座推薦やフォロー優先順位の理由説明
- PDF: 申込書や受講案内の下書きイメージ
- 翻訳: 多言語学習者や保護者対応の拡張要素として活用可能

## レビュー観点
- 業種担当者: 受講者フォローと講座運営の流れが自然か
- Web担当者: 「継続率向上」「運営負荷軽減」の訴求が立つか
- UI/UX: 受講者と講座の両軸が混線せず見えるか

## `staffing` への影響を出さない条件
- この計画は `education` 向けの用語、下層構成、導線見直しに限定する
- 派遣コックピットの完成済み体験は基準サンプルとして保持し、文言や導線を変更しない
- 共通化が必要でも、`education` 条件で閉じることを前提にする
