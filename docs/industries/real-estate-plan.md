# 不動産業種 横展開計画（初版）

## 体験コンセプト
- 一言で言うと: 物件提案から内見、契約、重要事項対応までを迷いなく進められる営業管理デモ
- 管理アプリとしての価値: 顧客、物件案件、内見進捗、契約書類を一気通貫で見せる
- AI/技術の価値: 顧客条件に合う物件提案理由、契約FAQ、重要事項説明まわりの支援を自然に見せる

## まず検討するページ
- ダッシュ: 物件提案数、内見調整数、契約見込みを主役にする
- 顧客一覧: 検討温度感、希望条件、次回接触予定をカードで比較できるようにする
- 物件案件一覧: 地域、価格帯、内見可否、契約ステータスが一目で分かる構成にする
- 物件提案: なぜこの物件を薦めるかを、条件一致・生活導線・予算で説明する
- 契約書類: 重要事項説明、申込書、契約関連書類の進捗を見せる
- 物件ナレッジ: よくある質問、設備条件、入居ルールを参照できる構成にする

## タイトル・カード方針
- `dashboardTitle`: 「不動産ダッシュボード」を起点に、営業色よりも提案管理色を強める
- `labels.candidate`: 見込み顧客 / 顧客候補 のどちらが自然か要確認
- `labels.client`: 物件案件 から、物件 / 募集物件 / 提案物件 のどれが最適か比較する
- 一覧カードの主タイトル:
  - 顧客一覧: 検討ステージ + 希望エリア
  - 物件一覧: 物件名 + 賃料帯 + 内見可否

## 下層ページの中身方針
- 顧客詳細: 家族構成、予算、希望条件、通勤動線、接客メモを上位表示
- 物件詳細: 間取り、立地、設備、内見履歴、契約留意点を整理
- 書類ページ: 重要事項説明、申込書、本人確認の不足を早く把握できる構成にする
- ナレッジページ: 「初期費用」「保証会社」「ペット可否」「短期解約違約金」など、営業が即答したい FAQ を主軸にする

## 次アクション導線
- ダッシュ → 顧客一覧 → 物件提案 → 契約書類
- 物件提案 → 物件詳細 or 内見調整
- 契約書類 → 不足資料回収 or 契約確定
- モバイルでは「次に連絡すべき顧客」「次に進めるべき案件」がすぐ分かることを優先する

## 技術デモの活用
- ナレッジAI: 物件FAQ、初期費用説明、契約時の注意点
- マッチングAI: 顧客条件に合う物件の提案理由
- PDF: 申込書・契約関連の下書きや送付イメージ
- OCR: 本人確認書類や申込情報の読み取り補助に意味づけ可能
- 翻訳: 外国籍入居者対応の拡張ストーリーとして有効

## レビュー観点
- 業種担当者: 顧客と物件の情報の優先順位が営業現場に合っているか
- Web担当者: 「提案の速さ」「契約対応の安心感」が訴求軸として立つか
- UI/UX: 顧客一覧と物件一覧で比較軸が明確か、モバイルでも提案フローが迷わないか

## `staffing` への影響を出さない条件
- この計画は `real-estate` 向けの物件提案、契約書類、FAQ 導線の変更に限定する
- 完成済みの派遣コックピットは変更しない
- 共通ファイルを触る場合も、`real-estate` のみへ効く変更に閉じる
