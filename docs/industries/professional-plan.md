# 士業業種 横展開計画（初版）

## 体験コンセプト
- 一言で言うと: 案件進行、証憑管理、論点整理、顧客対応をまとめて見せる士業業務デモ
- 管理アプリとしての価値: 依頼者ごとの進行、証憑回収、申請/相談タスク、対応履歴が追える
- AI/技術の価値: FAQ、論点要約、申請書類下書き、案件相性の理由付けを支援として見せる

## まず検討するページ
- ダッシュ: 未回収証憑、対応期限、進行中案件、受任見込みを見せる
- 顧客一覧: 依頼内容、進捗、必要証憑、優先度をカードで比較する
- 案件一覧: 申請・相談テーマ、期限、担当者、必要対応を見せる
- マッチング: 担当者アサインや案件優先の理由を説明する
- 書類: 証憑回収、申請書類、差戻し対応を主役にする
- ナレッジ: 制度 FAQ、必要書類、申請手順、注意点を整理する

## タイトル・カード方針
- 相談案件、受任案件、申請案件など、現場の言葉として何が自然か確認する
- 顧客一覧カードでは「論点」「期限」「不足証憑」を優先表示する
- 案件一覧カードでは「案件種別」「進行段階」「次対応」を優先表示する

## 下層ページの中身方針
- 顧客詳細: 相談背景、論点、必要証憑、過去対応履歴を上位表示
- 案件詳細: 期限、進行ステップ、提出先、差戻しリスクを整理
- 書類ページ: 不足証憑、差戻し理由、再提出フローを分かりやすくする
- ナレッジページ: 「必要証憑」「提出期限」「要件該当性」などを中心にする

## 次アクション導線
- ダッシュ → 顧客一覧 → 案件一覧 → 書類 → 次対応
- ナレッジ → 案件詳細 or 書類ページ
- モバイルでは「今日中に返すべき案件」「不足証憑のある案件」を先に見せる

## 技術デモの活用
- ナレッジAI: 制度や手順 FAQ に最も相性が良い
- PDF: 申請書類や提出書類の下書き支援
- OCR: 受領証憑の読み取り補助
- マッチングAI: 担当者アサインや案件優先順位づけの説明

## レビュー観点
- 業種担当者: 案件進行と証憑管理の実務感があるか
- Web担当者: 「ミス削減」「進行可視化」の訴求が立つか
- UI/UX: 論点・期限・不足書類の優先順位が明快か

## `staffing` への影響を出さない条件
- この計画は `professional` 向けの案件進行、証憑管理、FAQ の見せ方に限定する
- 派遣コックピットの完成済み体験は変更しない
- 共通ファイルに触れる場合は `professional` のみの条件分岐、または `staffing` 既定値維持で対応する
