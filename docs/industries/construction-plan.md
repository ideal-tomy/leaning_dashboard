# 建設・現場業種 横展開計画（初版）

## 体験コンセプト
- 一言で言うと: 配員、安全書類、入場準備、現場進行を止めないための現場管理デモ
- 管理アプリとしての価値: 現場ごとの配員状況、安全書類、入場可否、日次オペレーションをまとめて見せる
- AI/技術の価値: 配員候補の理由説明、必要書類の不足把握、現場ルールの FAQ 支援を見せる

## まず検討するページ
- ダッシュ: 稼働現場数、未提出安全書類、本日入場予定、要確認案件を主役にする
- 候補者一覧: 資格、経験工種、入場可否、安全教育状況が分かるカードにする
- 案件一覧: 現場名、工期、必要資格、入場条件、元請け制約を見せる
- マッチング: 資格一致、安全教育、現場経験を理由として説明する
- 書類: グリーンファイル、安全書類、資格証、入場申請の進捗を見せる
- 実務: 当日朝の入場調整、欠員補充、現場報告が回るイメージにする

## タイトル・カード方針
- `dashboardTitle`: 「現場ダッシュボード」は維持しつつ、工期・配員・安全を感じるコピーへ寄せる
- `labels.client`: 現場案件 / 現場 のどちらが営業・現場双方に通じるか確認する
- `labels.documents`: 安全書類 / 入場書類 のどちらを主ラベルにするか確認する
- 一覧カードの主タイトル:
  - 候補者一覧: 氏名 + 工種 + 安全教育ステータス
  - 現場一覧: 現場名 + 工期 + 必須資格

## 下層ページの中身方針
- 候補者詳細: 保有資格、現場経験、入場教育、健康面の留意点を上位表示
- 現場詳細: 現場ルール、入場条件、持込物、工程の重要日を整理
- 書類ページ: 足場、高所、重機、KY など現場固有の提出物を明示する
- ナレッジページ: 「安全帯」「新規入場者教育」「提出期限」「元請けルール」など現場で詰まりやすい FAQ を軸にする

## 次アクション導線
- ダッシュ → 候補者一覧 / 現場一覧 → マッチング → 安全書類 → 実務
- マッチング → 配員確定 → 入場書類確認
- 実務 → 当日不足対応 → 現場報告
- モバイルでは「今止まりそうな現場」が一目で分かることを優先する

## 技術デモの活用
- マッチングAI: 資格、経験、安全教育の合致理由を説明する主役デモ
- ナレッジAI: 元請けルール、安全教育、書類期限の FAQ に活用
- PDF: 入場関連書類や提出パックの下書きイメージ
- OCR: 資格証や本人確認資料の読み取り補助
- 翻訳: 外国籍作業員との安全伝達支援として有効

## レビュー観点
- 業種担当者: 配員判断に必要な情報が過不足なく見えるか
- Web担当者: 「現場を止めない」「安全書類に強い」という訴求が立つか
- UI/UX: モバイルでも朝の判断に使えるか、PC では複数現場比較がしやすいか

## `staffing` への影響を出さない条件
- この計画は `construction` 向けの文言・導線・カード内容の変更に限定する
- 完成済みの派遣コックピット（`staffing`）のタイトル、下層ページ、技術デモ構造は変更しない
- 共通ファイルを触る場合も、`construction` 条件で分岐するか、`staffing` の既定値を維持する
