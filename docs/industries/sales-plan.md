# 営業業種 横展開計画（初版）

## 体験コンセプト
- 一言で言うと: 商談進行、提案先優先度、受注見込みを整理して、次の一手が明確な営業デモ
- 管理アプリとしての価値: 見込み顧客、案件、提案状況、受注確度、資料対応が一目で分かる
- AI/技術の価値: 次に提案すべき理由、競合比較、FAQ、提案資料下書き支援を見せる

## まず検討するページ
- ダッシュ: 商談数、受注見込み、停滞案件、要フォロー案件を見せる
- 顧客一覧: 業種、検討度、課題、次回接点をカードで比較する
- 案件一覧: 提案テーマ、フェーズ、金額感、競合状況を見せる
- マッチング: 顧客課題と提案内容の一致理由を説明する
- 書類: 提案資料、見積、契約ドラフトの進捗を見せる
- ナレッジ: 提案FAQ、競合比較、トーク例、想定反論を整理する

## タイトル・カード方針
- `labels.candidate` は見込み顧客、`labels.client` は提案案件寄りに整理する
- 顧客カードでは「業種」「検討度」「主要課題」を主役にする
- 案件カードでは「提案テーマ」「受注確度」「次回アクション」を主役にする

## 下層ページの中身方針
- 顧客詳細: 直近会話、意思決定者、提案課題、競合状況を上位表示
- 案件詳細: 提案内容、想定効果、競合差分、次回商談準備を整理
- 書類ページ: 見積、提案書、NDA 前後で出せる資料の整理を重視する
- ナレッジページ: 「競合との差分」「提案時の想定質問」「断り文句への返し」などを主軸にする

## 次アクション導線
- ダッシュ → 顧客一覧 → 案件一覧 → ナレッジ / 書類
- マッチング → 商談準備 → 提案資料確認
- モバイルでは「今日接触すべき顧客」と「止まっている案件」をすぐ見せる

## 技術デモの活用
- ナレッジAI: 営業 FAQ、競合比較、提案トーク支援
- マッチングAI: 提案先優先順位や提案理由の説明
- PDF: 見積や提案資料の下書きストーリー
- 翻訳: 海外顧客や多言語対応の拡張要素として有効

## レビュー観点
- 業種担当者: 商談の進め方と案件管理の順序が自然か
- Web担当者: 「受注率向上」「営業の再現性向上」が訴求できるか
- UI/UX: 顧客、案件、提案理由の切り替えが分かりやすいか

## `staffing` への影響を出さない条件
- この計画は `sales` 向けの商談進行、提案理由、提案資料導線に限定する
- 派遣コックピットの既存体験は変更しない
- 共通ファイル変更時は `sales` 向け条件分岐または `staffing` 既定値維持を前提にする
