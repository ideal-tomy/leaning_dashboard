# 実行計画 — 外国人材支援・EラーニングSaaS（デモ → MVP）

## この文書の目的

- **営業デモ（P0）→ 契約後MVP（P1）→ 差別化（P2）→ 拡張（P3）** を迷わず進めるための**全体マップ**とする。
- **Cursor の PLAN モード**でエピック単位の詳細PLANを起こし、承認後に **Agent モード（build / 実装）** で実行する**繰り返し可能な型**を固定する。

---

## Cursor での推奨ワークフロー（毎サイクル同じ）

1. **PLAN モード**を開く。
2. 下記「エピック別：PLANモード用プロンプト」を**そのまままたはコンテキストを足して**貼り、**そのエピックだけ**の実装計画を出力させる。
3. 出力PLANをレビューし、スコープ・ファイルパス・完了条件を確定する。
4. **Agent モード**に切り替え、確定PLANに従い実装・ビルドする。
5. エピック末尾の **Definition of Done（完了条件）** を満たしたら次エピックへ。

### Agent モードでの共通指示（ビルド確認）

実装タスクの最後には、原則として次を含める（プロジェクトに合わせてコマンドは読み替える）。

```text
変更後、プロジェクトルートで `npm run build` を実行し、エラーがあれば修正してから完了とする。
```

### エレベーターピッチ（1文）

**主聴衆である受入企業（工場）に対し、登録支援機関が取る管理費の「中身」として、外国人スタッフの日本語・職場倫理の学習進捗とエビデンスをダッシュボードで見せるデモを行う。一方で、ビザ申請の本番API連携・行政システムへの送信・LINE本番・会計の完全自動連携はスコープに含めない。**

### ペルソナと週次利用

| 項目 | 内容（各1行） |
|------|----------------|
| **主ペルソナ** | 登録支援機関の運用担当者（オペ）：外国人スタッフのフォロー、工場との連絡、面談・報告の下準備を担う。 |
| **週1で見る画面（主ペルソナ）** | 誰の学習・期限対応が止まっているか、工場向けに今週提示できるサマリーが一覧で分かる **Admin（支援機関）向けトップ／ダッシュボード**。 |

### デモで見せる順序（Client → Admin → Worker）

1. **Client（工場・受入企業）**  
   **見せる一言:** 「月2〜3万円の管理費で、スタッフの日本語・倫理研修の進捗と修了状況が、数字とバッジで説明できます。」  
   成長グラフ・修了イメージ・月次レポート導線（プレースホルダ可）を見せる。

2. **Admin（支援機関・紹介会社）**  
   **見せる一言:** 「全スタッフの進捗・期限アラート・拠点別の集計をここで把握し、工場に渡す説明資料のたたき台にします。」  
   一覧・フィルタ・キラーカード（進捗停滞検知など）の中枢を見せる。

3. **Worker（外国人スタッフ）**  
   **見せる一言:** 「スマホで母国語サポート付きの学習とリマインドができ、工場向け画面の数値とつながるイメージです。」  
   モバイル幅UI・言語切替のイメージを軽く見せる（デモは同一アプリ内の表示モード切替想定）。

### やらないこと（スコープ外・デモ）

1. **実ビザ申請API／在留申請システムとの本番連携** — デモは「申請書類パック生成」の **トースト／モーダル** または固定テンプレの見せ場まで。
2. **行政（出入国在留管理庁等）への定期報告・届出の自動送信** — 既存事務SaaSの領域として切り分け、本プロダクトのデモでは扱わない。
3. **LINE Messaging API の本番配信・利用者IDの本番紐付け** — デモは文言・ボタン・通知イメージに留める。
4. **会計ソフト・請求書発行の完全自動連携** — デモはCSV／PDF **イメージ** またはプレースホルダ。本番連携はP2以降で検討。
5. **賃金・懲戒に直結する「評価スコア」の正式運用** — 研修修了表示と分離。現場評価を数値化して見せる場合は **※要合意**（労務・契約とセット）。

### 既存SaaSとの棲み分け

既存の登録支援系SaaSは、求人・特定技能などの **書類作成・手続き・定期報告の事務効率化** が主価値である。当デモプロダクトはそれと競合させず、**日本語・職場倫理などEラーニングの進捗と理解度、フォロー・面談の記録の見える化、工場に渡す説明用サマリー・レポート** にフォーカスする。将来的なデータ連携はエクスポートやAPIを想定し、デモでは **「事務は既存ツール、育成と証跡はこちら」** の役割分担を話術で明確にする。
